# CLAUDE.md

Guidance for working in this repo.

## What this is

`icon-api` - an Axum HTTP service that resolves file/folder icons (given a path,
returns an icon id). Cargo workspace, Rust edition 2021, toolchain 1.92.

## Layout

- `services/api/server` - the `icon-api` binary crate (Axum). Module tree:
  `src/icon_api/core/v1/icons/{db,memory,history}.rs`.
- `libs/iconator` - given fst lookup tables (`icons/lookup_table.rs`) + `get_icon_for_file/folder`.
- `libs/postgres` - Diesel models, `schema.rs` (schema `icons`), pooled connections.
- `libs/redis` - deadpool-redis pool.
- `libs/types` - shared request/response DTOs (the API contract; used by server **and** sdk).
- `libs/sdk` - typed async client (`IconClient`).
- `libs/telemetry`, `libs/utils` - metrics/tracing, secrets/db helpers.
- `db/migrations` - schema + icon seed (generated by `scripts/gen_icon_seed.py`).

## Endpoints (`/api/icons/v1`)

`GET /icons/file|folder?path=<path>` (Postgres+Redis), `GET /icons/memory/file|folder?path=<path>`
(in-memory fst), `GET /icons/history`. No match -> `200` with `iconId: null`.

## Commands

- `make test` (nextest, workspace), `make lint` (fmt-check, clippy `-D warnings`, machete, prettier, markdown, typos)
- `make fmt` before committing (rustfmt `max_width = 80`, taplo for TOML, prettier for md/json)
- Run: `cargo run --bin icon-api` (needs `.env`; `cp .env.sample .env`)

## Conventions & gotchas

- **rustfmt `max_width = 80`** - run `cargo fmt` after edits or CI's fmt check fails.
- **clippy runs with `-D warnings`** across the whole workspace - no warnings allowed.
- Icon data has **one source of truth**: `libs/iconator/icons/lookup_table.rs`. If it
  changes, regenerate the DB seed with `python3 scripts/gen_icon_seed.py db/migrations/2025-01-01-000003_seed_icons/up.sql`.
- DB and in-memory paths must return identical results - keep resolution logic in
  `icon_api/core/v1/icons/resolve.rs` in sync with `iconator`.
- Postgres config is `DATABASE_URL` (+ optional `DATABASE_RO_URL`); `POSTGRES_*` are
  the source of truth the container/exporter use.
- Startup is resilient: the service boots even if Postgres/Redis are down (in-memory
  endpoints keep working, DB endpoints return `503`).
